## 一、Pretrain

Pretrain，即预训练，是训练LLM 最核心也是工程量最大的一步，使用海量无监督文本对随机初始化的模型参数进行训练。与传统预训练模型的核心差异在于预训练的体量和资源消耗，且往往需要使用更大规模的预训练语料。

### 1. 分布式训练框架

数据并行和模型并行。

#### 数据并行

训练模型的尺寸可以被单个GPU内存容纳。

> 在不同 GPU 和不同批数据上运行，每一次前向传递完成之后，收集所有实例的梯度并计算梯度更新，更新模型参数之后再传递到所有实例。也就是在数据并行的情况下，每张 GPU 上的模型参数是保持一致的，训练的总批次大小等于每张卡上的批次大小之和。

#### 模型并行

但是，当 LLM 扩大到上百亿参数，单张 GPU 内存往往就无法存放完整的模型参数。

因此可以将模型拆分到多个 GPU 上，每个 GPU 上存放不同的层或不同的部分，从而实现模型并行。

### 2. 预训练数据的处理与清洗

#### 1. 文档准备：

 URL 过滤（根据网页 URL 过滤掉有害内容）、文档提取（从 HTML 中提取纯文本）、语言选择（确定提取的文本的语种）等。‘

#### 2. 语料过滤

去除低质量、无意义、有毒有害的内容。

**两种方法：**

- 基于模型的方法，即通过高质量语料库训练一个文本分类器进行过滤；
- 基于启发式的方法，一般通过人工定义 web 内容的质量指标，计算语料的指标值来进行过滤。

#### 3. 语料去重

删除训练语料中相似度非常高的文档，也是必不可少的一个步骤。去重一般基于 hash 算法计算数据集内部或跨数据集的文档相似性，将相似性大于指定阈值的文档去除；也可以基于子串在序列级进行精确匹配去重。

## 二、SFT

只通过预训练的LLM就像一个死板背书的书生，在没有进一步微调前，无法与其他下游任务进行适配。因此需要 SFT (Supervised Fine-Tuning，有监督微调)。面对能力强大的LLM，我们往往不再是在指定的下游任务上构造有监督数据进行微调，而是选择训练模型的“通用指令遵循能力”。

### 1. 指令微调

我们训练的输入是各种类型的用户指令，而需要模型拟合的输出则是我们希望模型在收到该指令后做出的回复。

```
input:告诉我今天的天气预报？
output:根据天气预报，今天天气是晴转多云，最高温度26摄氏度，最低温度9摄氏度，昼夜温差大，请主义保暖喔。
```

SFT 的主要目标是让模型从多种类型、多种风格的指令中获得泛化的指令遵循能力，也就是能够理解并回复用户的指令。

### 2. 多轮对话

```
 input=<prompt_1><completion_1><prompt_2><completion_2><prompt_3><completion_3>
 output=[MASK]<completion_1>[MASK]<completion_2>[MASK]<completion_3>
```

## 三、RLHF

RLHF，(Reinforcement Learning from Human Feedback，即人类反馈强化学习)，利用强化学习来训练LLM的关键步骤。

从功能上出发，我们可以将LLM的训练过程分成预训练与对齐两个阶段，预训练的核心作用是赋予模型海量的知识。而对齐是让模型与人类价值观一致，从而输出人类希望其输出的内容。

STF是让LLM和人类的指令对起，从而具有指令的遵循能力。而GLHF是从深层次令LLM和人类价值观对齐，令其达到安全、有用、无害的核心标准。

### 1. 思路

引入强化学习的技术，通过实时的人类反馈令 LLM 能够给出更令人类满意的回复。

### 2. 两大步骤

#### 1. RM -- Reward Model

对于LLM的每一个回复，RM进行打分，这个打分反映的是生成的回复符合人类偏好的程度。

#### 2. PPO -- Proximal Policy Optimization

近端策略优化算法。
